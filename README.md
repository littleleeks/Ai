# Claude Code Driver Security Analyzer

Professional Windows/Linux kernel driver security auditing command for Claude Code.
Supports automatic driver code review .

This tool helps security researchers, reverse engineers, and driver developers quickly identify memory safety issues, IOCTL design flaws, privilege escalation risks, and unsafe kernel API usage.

## ✨ Features

## 🔍 Automatic Driver Security Analysis

- Windows kernel driver auditing (ntddk.h, DriverEntry, NTSTATUS)
- Linux kernel driver auditing (__init, kernel module patterns)
- Memory safety vulnerability detection
- IOCTL interface security checks

## 🧠 Advanced Security Checks

The analyzer reviews driver code across multiple security domains:
- Memory Safety
- Input Validation
- Concurrency & Race Conditions
- Privilege & Access Control
- Dangerous API Usage
- IOCTL Interface Design
- Kernel Resource Management


## 📊 Structured Security Report

Automatically generates a professional Markdown audit report including:

- Risk level classification

- Vulnerability table

- Unsafe API usage

- Design flaws

- Remediation recommendations

## Installation

Add the command to your Claude Code commands directory.

Example:
```
.claude/commands/analyze-driver.md
```

Then paste the command definition.

## Command Definition
```
---
description: 专业驱动安全审计（支持代码分析和MCP工具联动）
argument-hint: <驱动源代码或 "包含MCP字段的语句">
allowed-tools:
- Read
- Write
- WebSearch
- MCP
---
```
## Usage
### 1️⃣ Driver Code Analysis

Analyze Windows or Linux kernel driver source code.

Example:
```
/driver-analyzer
#include <ntddk.h>

NTSTATUS DriverEntry(PDRIVER_OBJECT DriverObject,
                     PUNICODE_STRING RegistryPath)
{
    ...
}
```
The command will automatically:
- Detect driver patterns
- Perform security analysis
- Generate a structured vulnerability report

### 2️⃣ MCP-assisted Analysis

MCP automatically analyzes in conjunction with tools such as IDA and XDBG
```
Example:

/driver-analyzer 根据MCP连接对代码进行分析
```

## Security Audit Coverage

The analyzer checks the following areas.

### Memory Safety

- Unsafe buffer operations (memcpy, strcpy)
- Integer overflow in memory allocation
- Use-after-free
- Incorrect pool allocation
```
Example checks:

ExAllocatePool / ExFreePool mismatch
RtlCopyMemory without size validation
```
### Input Validation
- User pointer validation
- IOCTL buffer length verification
- Unsafe string operations
```
Example checks:

ProbeForRead / ProbeForWrite
access_ok
```
### Concurrency Issues

- Missing locks
- Incorrect IRQL lock usage
- Reference counting issues
```
Example checks:

spinlock misuse
TOCTOU race conditions
```

### Privilege & Access Control

- Device object security descriptor misconfiguration
- Symbolic link exposure
- User-controlled function pointers

### Dangerous Kernel APIs

Detects risky kernel APIs such as:
```
MmMapIoSpace
ZwSystemDebugControl
KeSetAffinityThread
```
Also warns about deprecated APIs:
```
ExAllocatePool → should use ExAllocatePool2
```
### IOCTL Security
Checks for common IOCTL vulnerabilities:
- METHOD_NEITHER unsafe pointer handling
- Incorrect buffer type assumptions
- Abusable IOCTL codes

### Kernel Resource Management
- Handle leaks
- IRP completion issues
- Work item cleanup problems

### Safety Design

The command is designed with strict safety guarantees.

## No Code Execution

All inputs are treated as text for analysis only.

The command will never execute driver code.

### Input Validation
- Rejects binary inputs
- Detects overly large code (>200KB)
- Supports segmented analysis

### Tool Permission Control

Only the following tools are used:
```
Read
Write
WebSearch
MCP
```
No unauthorized operations will be attempted.
